# 剧本资产拆解（Script Asset Breakdown）

> 将短剧/影视分镜脚本 PDF 自动拆解为 **场景卡 + 角色卡**，合并细碎镜头为可制作大场景，并为每个场景/角色生成 AI 提示词用于影视概念图、分镜气氛图和置景参考图。

## 触发条件

当用户提到以下任意关键词时触发：
- "剧本资产拆解" / "拆解剧本" / "拆解脚本" / "拆解分镜"
- "场景卡" / "角色卡"
- "剧本拆解" / "分镜拆解"
- 上传了分镜脚本 PDF 并要求整理/分析

## 工作流程

### 第 1 步：提取脚本内容

使用 `pdfplumber` 提取 PDF 全文，保留所有场次/镜号/台词/动作描述。

```bash
python3 -c "
import pdfplumber
with pdfplumber.open('<PDF路径>') as pdf:
    for i, page in enumerate(pdf.pages):
        text = page.extract_text()
        if text:
            print(f'PAGE {i+1}:')
            print(text)
"
```

### 第 2 步：识别核心信息

从提取的文本中识别：

| 信息类别 | 提取内容 |
|----------|----------|
| **剧名** | 标题/文件名中的片名 |
| **类型/风格** | 题材类型、叙事风格、视觉基调 |
| **总时长** | 各镜号时长合计 |
| **剧情概要** | 开篇剧情概要段落 |
| **角色** | 所有出场角色及其身份、关系 |
| **镜号列表** | 每个镜号的编号、时长、景别、镜头运动、内容描述 |
| **空间** | 所有场景发生的物理空间 |
| **时间** | 每个场景的时间（白天/傍晚/夜晚/深夜） |
| **台词** | 所有对白和画外音 |
| **关键道具** | 反复出现或剧情关键的道具 |

### 第 3 步：合并细碎镜头为大场景

> **核心原则**：同空间 + 同时间段 → 合并为一个可制作大场景

合并规则：
1. **同一物理空间** — 即使原脚本切了多个镜号，只要在同一空间内，就合并
2. **时间连续或同一天** — 不跨天合并
3. **人物交集** — 同空间内人物自然交集则合并
4. **情绪单元完整** — 一个完整的情绪推进过程不拆散

反例（不要这样做）：
- 不要把"猫眼观察"和"敲门对峙"拆成两个场景——它们都在走廊/门口，时间连续
- 不要把"做饭"和"吃饭"拆开——都在俞浅家客厅/厨房，同一时间段

### 第 4 步：生成角色卡

每个角色生成一张卡，包含：

```markdown
### 角色 XX：角色名

| 维度 | 描述 |
|------|------|
| **身份** | 社会身份/角色定位 |
| **年龄** | 约 X-Y 岁 |
| **外貌特征** | 面容、体型、穿着风格、特殊标识 |
| **性格关键词** | 3-5 个精准形容词 |
| **人物弧光** | 从 → 到（角色变化轨迹） |
| **核心特质** | 一句话概括这个角色最独特的地方 |
| **道具关联** | 与该角色绑定的关键道具列表 |
```

**角色卡 AI 提示词规范**：
- 格式：`**影视角色概念图 | 角色名**`
- 内容：年龄、外貌、服装、姿态、表情、光线氛围、技术参数
- 标注用途：`角色定妆参考 / 影视概念图`

### 第 5 步：生成场景卡

每个大场景生成一张卡，包含：

```markdown
### 场景 XX：场景名称

| 维度 | 内容 |
|------|------|
| **原镜号** | 合并了哪些原始镜号 |
| **时长** | 该场景总时长 |
| **空间** | 物理空间描述 |
| **时间** | 白天/傍晚/夜晚/深夜 |
| **人物** | 出场角色列表 |
| **场景概述** | 1-2 句话概括 |
| **核心情节** | 用编号列出关键情节节点 |
| **情绪基调** | 核心情绪 + 一句话解读 |
| **美术置景要点** | 空间布局、光线、色调、关键陈设 |
```

**场景卡 AI 提示词规范**：
- 每个场景至少 1 条，复杂场景可拆分 2 条（如 A/B 两个关键画面）
- 格式：`**影视分镜气氛图 | 场景名**` 或 `**影视置景参考图 | 场景名**`
- 必须包含：构图角度、人物位置/姿态、光线来源与色调、镜头焦段、画幅比例、情绪氛围关键词
- 标注用途：`分镜气氛图 / 置景参考图`

### 第 6 步：输出统筹表

#### 场景调度总览

```markdown
| 场景编号 | 场景名称 | 空间 | 时间 | 时长 | 核心人物 | 情绪关键词 |
|:---:|------|------|------|:---:|------|------|
```

#### 色调演进建议

根据剧情发展，规划色调从 → 到 → 的变化：

| 阶段 | 场景范围 | 主色调 | 含义 |
|------|----------|--------|------|
| 第一幕 | 场景XX-XX | 色调描述 | 情感含义 |
| 第二幕 | 场景XX-XX | 色调描述 | 情感含义 |
| 第三幕 | 场景XX-XX | 色调描述 | 情感含义 |

#### 核心道具清单

| 道具 | 所属角色 | 出现场景 | 美术要点 |
|------|----------|----------|----------|
| 道具名 | 角色名 | 场景编号 | 材质/颜色/形态描述 |

### 第 7 步：输出为 Markdown 文件

将完整拆解结果写入 `/workspace/《剧名》场景卡与角色卡.md`。

## AI 提示词撰写规范

### 影视概念图 / 角色定妆图

必须包含：
1. 人物年龄、面容特征、发型、服装
2. 身体姿态与动作
3. 所在环境（空间与光线）
4. 面部表情与情绪状态
5. 构图与景别
6. 技术参数：`电影级写实风格，4K画质，35mm镜头` 或 `广角/长焦`
7. 色调与氛围关键词

### 分镜气氛图

必须包含：
1. 场景空间与时间（光线来源与色温）
2. 人物在画面中的位置与姿态
3. 构图角度（俯拍/仰拍/平视/过肩/猫眼视角等）
4. 镜头焦段与画幅比例（`宽银幕`/`35mm`/`广角微畸变`）
5. 色调对比与情绪氛围关键词
6. 景深控制（`浅景深`/`大景深`）

### 置景参考图

必须包含：
1. 空间类型与面积感
2. 墙面/地面/天花材质与颜色
3. 主要家具与陈设
4. 光线来源（窗户/灯具）
5. 色调与风格关键词
6. 可补充细节道具清单

## 质量标准

拆解完成后自检：
- [ ] 所有镜号都被合并到某个大场景中，无遗漏
- [ ] 每个出场角色都有角色卡
- [ ] 每个场景卡都包含 AI 提示词
- [ ] AI 提示词均包含用途标注
- [ ] 场景调度总览表、色调演进建议、核心道具清单均完整
- [ ] 文件输出到 `/workspace/` 目录
